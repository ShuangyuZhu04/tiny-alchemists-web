Unity NGO Integration (Netcode for GameObjects)

Architected the entire multiplayer infrastructure using Unity's Netcode for GameObjects (NGO) framework. Set up the NetworkManager, configured transport layers, and established the connection flow for host/client topology. Implemented reliable NetworkVariable synchronization for shared game state including cauldron contents, potion inventories, and player scores across all connected clients.

Unity NGO NetworkManager NetworkVariable C#
🛡️

Server-Authoritative Game Logic

Designed and implemented a fully server-authoritative architecture to ensure fair and deterministic gameplay for all players. All critical actions — ingredient submission, potion brewing validation, and cauldron explosion triggers — execute exclusively on the server via ServerRpc calls. Client-side prediction is used only for non-critical visual feedback, while ClientRpc broadcasts propagate authoritative state changes to every connected player in real time.

ServerRpc ClientRpc Fair Play Authority Model
🧪

Status Effect Manager & Potion System

Built the complete potion effect pipeline using a modular BaseEffect / ModifierEffect architecture. Five potion types (Fire, Ice, Magnetic, Bouncy, Gravity) each implement unique physics-based behaviors via a shared interface. Effects are composed at brew-time from base ingredients and modifiers, serialized over the network using NetworkList<int>, and instantiated on all clients through a factory pattern that maps ingredient IDs to concrete effect classes.

Status Effects Factory Pattern NetworkList Physics
💥

Cauldron Explosion & Player Interaction Systems

Implemented the signature cauldron explosion mechanic: a server-side timer tracks how long a player remains inside the cauldron trigger volume. When the threshold is reached, an explosion force launches the player via the existing PlayerPush RPC — a core party-game moment that always gets laughs. Also built the complete player interaction system (PlayerInteract) with raycast-based pickup, throw mechanics, and the IInteractable interface for extensible world interactions.
